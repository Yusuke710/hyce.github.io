LLM as HPC Expert: Extending RAG Architecture for HPC Data

SCA2025

Yusuke Miyashita1, Patrick Kin Man Tung1, Johan Barthélemy2
1University of New South Wales, 2NVIDIA
Paper Poster Code

People often face a steep learning curve when using HPC systems, such as understanding the structure of the HPC cluster and mastering the commands required to navigate it.

Consequently, users frequently direct their queries to the help desk, which can quickly become overwhelmed with repetitive questions. Maintaining a team of HPC experts for 24/7 support is also costly and inefficient.

An ideal solution is to replace the help desk with an AI system capable of operating 24/7 while providing fast responses. Retrieval-Augmented Generation (RAG) systems, which integrate proprietary data with large language models (LLMs), are a promising option. However, for HPC systems, simply feeding documents to a RAG system is insufficient since many queries require real-time execution of commands to retrieve specific cluster information. The challenge, therefore, is: how can we effectively combine RAG with command-line interfaces (CLI)?

Solution: Hypothetical Command Embeddings (HyCE)

Our proposed system enhances traditional RAG architectures by incorporating Hypothetical Command Embeddings (HyCE). This approach retrieves command descriptions based on user queries, executes the appropriate commands, and integrates the resulting output into the LLM’s context. By combining RAG with real-time command execution, HyCE enables precise and efficient answers tailored to HPC-specific queries.

Thought Process Behind HyCE

  • Naive Approach: Run all commands and index their outputs for retrieval. This method is inefficient since executing all possible commands for every query is time-consuming.
  • Improved Approach: Embed each command for retrieval and execute only the retrieved commands in real time.
  • HyCE: Instead of embedding raw command names, HyCE embeds detailed command descriptions. When a relevant description is retrieved, the corresponding command is executed and the command outputis included in the context of the LLM, ensuring both efficiency and accuracy.
